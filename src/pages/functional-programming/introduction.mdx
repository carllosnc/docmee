# Functional Programming Concepts

Functional programming is a programming paradigm where programs are constructed by applying and composing functions. It is a declarative programming paradigm in which function definitions are trees of expressions that map values to other values, rather than a sequence of imperative statements which update the running state of the program.

## Core Concepts

### Pure Functions
A function where the return value is only determined by its inputs, without observable side effects. This makes functions predictable and easier to test.

### Immutability
In functional programming, state is not modified after it's created. Instead of changing data, you produce new data with the desired changes.

### First-Class and Higher-Order Functions
Functions are treated as first-class citizens, meaning they can be bound to names, passed as arguments, and returned from other functions. Higher-order functions are functions that take other functions as arguments or return them.

### Declarative vs Imperative
Functional programming focuses on describing *what* to do rather than *how* to do it.

### Recursion
Instead of using traditional loops (like `for` or `while`), functional programming heavily relies on recursion for iterating over data structures or changing state over time.
